# BerthCare Icon Library

## Overview

The BerthCare icon library provides a comprehensive set of healthcare-specific icons optimized for both mobile and web interfaces. All icons follow consistent design principles, maintain accessibility standards, and support multiple sizes and states.

## Design Principles

### Healthcare Context
- **Medical Clarity:** Icons clearly represent medical concepts and actions
- **Universal Recognition:** Uses internationally recognized medical symbols
- **Cultural Sensitivity:** Appropriate for diverse healthcare environments
- **Professional Standards:** Aligns with healthcare industry conventions

### Technical Requirements
- **Scalable:** Vector-based for crisp rendering at all sizes
- **Accessible:** Meets WCAG 2.1 AA contrast requirements
- **Consistent:** Uniform stroke width, corner radius, and style
- **Platform Optimized:** Native look and feel on iOS and Android

## Icon Specifications

### Size System
```
- Small: 16px (inline with text)
- Default: 24px (standard UI elements)
- Large: 32px (primary actions)
- Touch Target: 44px minimum (invisible hit area)
```

### Color Palette
```
- Primary: #1B4F72 (interactive elements)
- Success: #239B56 (positive status)
- Warning: #DC7633 (attention needed)
- Error: #CB4335 (critical status)
- Neutral: #495057 (inactive states)
- Muted: #6C757D (secondary information)
```

### States
```
- Default: Standard color and opacity
- Hover: 10% darker, slight scale (1.05x)
- Active: 15% darker, pressed effect
- Disabled: 40% opacity, no interaction
- Loading: Animation or spinner overlay
```

## Accessibility Guidelines

### Screen Reader Support
- All icons include meaningful alt text
- Decorative icons marked with `aria-hidden="true"`
- Interactive icons have accessible names
- Status changes announced to screen readers

### High Contrast Mode
- Icons maintain visibility in high contrast
- Outline versions for better contrast
- Alternative text descriptions for complex icons
- Color-blind friendly combinations

### Voice Control
- Icons have voice-friendly names
- Consistent naming patterns across similar functions
- Alternative voice commands for common actions

## Implementation Guidelines

### React Native
```jsx
import { Icon } from '@berthcare/icons';

// Basic usage
<Icon name="home" size={24} color="#1B4F72" />

// With accessibility
<Icon
  name="phone"
  size={32}
  color="#239B56"
  accessibilityLabel="Call patient"
  accessibilityRole="button"
/>

// Animated state
<Icon
  name="sync"
  size={24}
  animated={true}
  accessibilityLabel="Syncing data"
/>
```

### CSS/Web
```css
/* Icon base styles */
.icon {
  display: inline-block;
  width: var(--icon-size, 24px);
  height: var(--icon-size, 24px);
  fill: currentColor;
  transition: all 0.15s ease-in-out;
}

/* Size variants */
.icon--small { --icon-size: 16px; }
.icon--large { --icon-size: 32px; }

/* State variants */
.icon--interactive:hover {
  transform: scale(1.05);
  opacity: 0.8;
}

.icon--disabled {
  opacity: 0.4;
  pointer-events: none;
}
```

### SVG Optimization
- Optimized for web delivery (compressed)
- Consistent viewBox (0 0 24 24)
- Cleaned paths for smallest file size
- Icon fonts available for broader compatibility

## Brand Consistency

### Design Language
- Rounded corners (2px radius) for friendly appearance
- 2px stroke width for clarity at small sizes
- Consistent proportions across icon families
- Minimal detail for scalability

### Medical Accuracy
- Clinically accurate representations
- Reviewed by healthcare professionals
- Culturally appropriate symbols
- International standard compliance where applicable
